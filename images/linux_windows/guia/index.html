    Aquí tens la documentació completa del procés que hem seguit per unir l'Ubuntu al domini Windows Server 2022/2025, utilitzant mètodes moderns i segurs.

    Documentació: Unió d'Ubuntu a Active Directory (Windows Server 2022/2025)

    Aquesta guia detalla el procés tècnic per integrar un client Ubuntu en un domini de Windows Server mitjançant xifratge AES i el servei SSSD.

    1. Escenari i Requisits

    • Servidor (DC): Windows Server 2022/2025 (IP: 192.168.2.101, Domini: vdomin.local).
    • Client: Ubuntu (VirtualBox).
    • Objectiu: Autenticació d'usuaris del domini a la màquina Linux.

    2. Preparació al Windows Server (Crucial)

    Abans d'unir el client, cal preparar el compte d'usuari administrador per admetre xifratge modern, ja que per defecte molts comptes no tenen generades les claus AES.

    Activar xifratge AES (PowerShell)

    Executa aquestes comandes al controlador de domini per forçar la compatibilitat:

    # 1. Definir contrasenya de seguretat
$password = ConvertTo-SecureString "Admin123!" -AsPlainText -Force

# 2. Reset de contrasenya per generar claus AES
Set-ADAccountPassword -Identity Administrador -NewPassword $password

# 3. Habilitar tipus de xifratge AES al compte
Set-ADUser -Identity Administrador -KerberosEncryptionType AES128,AES256

    3. Automatització de la unió (Script Ubuntu)

    Hem utilitzat un script en Bash que gestiona la instal·lació de paquets, la configuració de Kerberos i la unió directa mitjançant adcli.

    Fitxer: unir_domini.sh

    #!/bin/bash
# Script per unir Ubuntu a AD amb suport AES

DOMAIN="vdomin.local"
REALM="VDOMIN.LOCAL"
DC_IP="192.168.2.101"
ADMIN_USER="Administrador"
ADMIN_PASS="Admin123!"

# [1] DNS
cat > /etc/resolv.conf << EOF
nameserver $DC_IP
search $DOMAIN
EOF

# [2] Instal·lació de paquets
apt-get update -qq
apt-get install -y realmd sssd sssd-tools adcli krb5-user ntpdate -qq
ntpdate -u $DC_IP

# [3] Configuració Kerberos (Només AES)
cat > /etc/krb5.conf << EOF
[libdefaults]
    default_realm = $REALM
    dns_lookup_realm = false
    dns_lookup_kdc = true
    rdns = false
    forwardable = yes
    default_tgs_enctypes = aes256-cts-hmac-sha1-96 aes128-cts-hmac-sha1-96
    default_tkt_enctypes = aes256-cts-hmac-sha1-96 aes128-cts-hmac-sha1-96
    permitted_enctypes = aes256-cts-hmac-sha1-96 aes128-cts-hmac-sha1-96
EOF

# [4] Unió al domini (Mètode LDAP-Passwd)
# S'utilitza --ldap-passwd per evitar l'error "Message stream modified"
echo "$ADMIN_PASS" | adcli join --verbose --domain="$DOMAIN" \
    --domain-controller="$DC_IP" --login-user="$ADMIN_USER" \
    --stdin-password --ldap-passwd

    4. Configuració del Servei SSSD

    SSSD actua com el pont entre el sistema local i Active Directory.

    Fitxer: /etc/sssd/sssd.conf

    [sssd]
domains = vdomin.local
config_file_version = 2
services = nss, pam

[domain/vdomin.local]
ad_domain = vdomin.local
krb5_realm = VDOMIN.LOCAL
realmd_tags = manages-system joined-with-adcli
cache_credentials = True
id_provider = ad
krb5_store_password_if_offline = True
default_shell = /bin/bash
ldap_id_mapping = True
use_fully_qualified_names = False
fallback_homedir = /home/%u
access_provider = ad

    Permisos i aplicació:

    sudo chmod 600 /etc/sssd/sssd.conf
sudo systemctl restart sssd
sudo pam-auth-update --enable mkhomedir

    5. Verificació del Funcionament

    • Verificació d'identitat: id Administrador -> Ha de retornar els IDs numèrics del domini.
    • Inici de sessió: su - administrador -> Ha de crear el directori /home/administrador.
    • Registre al servidor: Al Windows Server, l'equip ALUMNE-VIRTUALB apareix a la carpeta Computers.
